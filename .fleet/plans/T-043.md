# Plan: T-043 — reviewer upstream approval gating

Part of the 5-task stacked-PR vision stack from issue #289.
See `.fleet/plans/T-041.md` for full plan and gotchas.

**Depends on:** T-042

**Files:**
- `scripts/fleet/fleet-labels` — add `fleet:awaiting-upstream-review`
- `.claude/commands/role-sonnet-reviewer.md`
- `.claude/commands/role-opus-reviewer.md`

Reviewer detects `baseRefName != "master"`, looks up upstream PR by head ref:
- Upstream OPEN with `fleet:needs-fix`/`fleet:blocker`: add
  `fleet:awaiting-upstream-review`, comment, skip verdict.
- Upstream OPEN with `fleet:approved`: proceed with own-diff review.
- Upstream MERGED: per #287, merger re-targeted; proceed normally.
- Upstream not found / closed-not-merged: comment, skip, surface to human.

**Acceptance:** open downstream while upstream has `fleet:needs-fix`
→ `fleet:awaiting-upstream-review` applied, no `fleet:approved`. After
upstream re-approved, next reviewer pass proceeds normally.

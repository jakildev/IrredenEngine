## Approach

### Single-system safety check

In the registration path, after the trait derives `SystemAccess`,
if `concurrency == PARALLEL_FOR`:

- `SystemAccess.uses_entity_id == true && !has_tag<ParallelSafe>` →
  FATAL: `"System '<name>' has EntityId first param; cannot run
  PARALLEL_FOR without explicit ParallelSafe tag"`.
- `SystemAccess.is_batch_form == true && !has_tag<ParallelSafe>` →
  FATAL: `"System '<name>' uses batch-form tick (std::vector<T>&);
  cannot run PARALLEL_FOR without explicit ParallelSafe tag"`.

The opt-in via `ParallelSafe` tag is intentionally explicit — the
author has to acknowledge the risk.

### Parallel dispatch

In `executeSystem`, for each archetype node matching the system's
component set:

```cpp
if (system.concurrency == Concurrency::PARALLEL_FOR) {
    IR_PROFILE_BLOCK("PARALLEL_FOR dispatch", COLOR_SYSTEMS);
    IRJobs::parallelFor(0, node->length_, system.grainSize,
        [&](int begin, int end) {
            for (int i = begin; i < end; ++i) {
                system.tickFn(node, i);  // existing per-row dispatch
            }
        });
} else {
    // existing serial path
}
```

`grainSize` defaults to 512 if not specified. Tunable via an
optional arg on the `createSystem` / `registerSystem` spec.

### IR_ASSERT_MAIN_THREAD

```cpp
#ifdef NDEBUG
#define IR_ASSERT_MAIN_THREAD() ((void)0)
#else
#define IR_ASSERT_MAIN_THREAD()                                 \
    do {                                                        \
        if (!IRJobs::isMainThread()) {                          \
            IR_FATAL("'{}' called off main thread", __func__);  \
        }                                                       \
    } while (0)
#endif
```

## Acceptance criteria

- `perf_grid_matrix.sh` 262K cell: ≥2× UPDATE throughput at
  `worker_threads = hw-2` vs `0`. File the number in
  `docs/perf-reports/threading_phase2.md`.
- Three POC systems regress nothing in `IrredenEngineTest`.
- Registration FATAL fires on:
  - `PARALLEL_FOR` + `EntityId` first param + no `ParallelSafe`.
  - `PARALLEL_FOR` + batch-form + no `ParallelSafe`.
  Both unit-tested.
- `IR_ASSERT_MAIN_THREAD` fires when called from a worker
  thread. Unit-tested by spawning `IRJobs::run` invoking an
  instrumented manager entry.

## Gotchas

- `grainSize = 512` is a starting heuristic. Don't auto-tune in
  this PR — that's a Phase 6+ followup. Resist the urge.
- Determinism: use **chunked** partition (`[i, i+grainSize)` per
  worker), not work-stealing — preserves per-row order for
  `--auto-screenshot` reproducibility. enkiTS's `TaskSet` does
  this by default; if you switch to a custom scheduler, you have
  to enforce it.
- The three POC systems were picked because their tick lambdas
  have no `EntityId` and no manager calls. Don't port a fourth
  in this PR even if it looks safe — keep the diff reviewable.
- The `Concurrency` enum's `MAIN_THREAD` alias for `SERIAL` is
  semantically distinct: `MAIN_THREAD` cannot share a parallel
  group (T-224 enforces). `SERIAL` is a passive default. Don't
  collapse them.

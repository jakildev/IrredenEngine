## This PR: Lua port of perf_grid + parity gate — formal acceptance gate for #293

### Scope

Mirror `creations/demos/perf_grid/` in pure Lua and use it as the formal acceptance gate for
#293. The parity measurement decides whether the design needs revision before #293 closes.

### Deliverables

- New demo `creations/demos/lua_perf_grid/`:
  - `main_lua.cpp` is a thin C++ shim that boots the engine and runs `main.lua`.
  - `main.lua` defines the wave-animation component (Lua-typed, native columns), the
    wave-animation system, and the entire pipeline.
  - 64×64×64 voxel grid (≈262k entities), same wave-amplitude/wave-period config as the C++
    `perf_grid`, same render pipeline.
  - `CMakeLists.txt` defining the `IRLuaPerfGrid` target.
- Built with `fleet-build --target IRLuaPerfGrid`, run with `fleet-run IRLuaPerfGrid`.
- Captured per-tick timings (via existing `IRProfile` infrastructure with `profiling_enabled = true`
  in `config.lua`) for both demos. Document the C++-vs-Lua delta in `docs/design/lua-driven-ecs.md`
  retrospective.

### Acceptance criterion (parity gate)

Lua version's wave-animation system per-tick cost ≤ **1.5×** the C++ `SystemPeriodicIdlePositionOffset`
per-tick cost on the same hardware.

### Fail behavior

If the gate fails, this PR does **not** merge. Instead, amend the design doc PR (T-099) with the
corrective decision (LuaJIT migration, codegen-bound bodies, schemafied-only enforcement, etc.)
before #293 can close. The corrective decision becomes its own architect ticket.

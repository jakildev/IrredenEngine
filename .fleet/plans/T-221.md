## Approach

### Job module

1. Vendor enkiTS by copy (not git submodule — match `metal-cpp`,
   `luajit` precedent).
2. CMake: static lib `enkiTS`, wrap in `IRJob` target that
   re-exports headers.
3. `JobManager::start(workerCount)`:
   - Detect macOS P-core count via
     `sysctlbyname("hw.perflevel0.physicalcpu", ...)`; on Linux/WSL
     use `hardware_concurrency()`. Cap requested at detected.
   - Configure `enki::TaskSchedulerConfig` with the worker count.
   - Per-worker startup callback: `EASY_THREAD("worker N")` (symmetric
     to `EASY_MAIN_THREAD` at `engine/profile/src/cpu_profiler.cpp:60`)
     + seed `thread_local std::mt19937` from the worker id.
4. `IRJobs::parallelFor(begin, end, grainSize, fn)`: wraps
   `enki::TaskSet`. Block on completion via
   `m_scheduler.WaitforTask(...)`.
5. `IRJobs::isMainThread()`: compare current thread id against the
   id captured at `JobManager::start`. Used by
   `IR_ASSERT_MAIN_THREAD` in T-222.

### Access traits

The trait walks the lambda parameter pack at compile time via the
existing `InvocableWithComponents` / `InvocableWithEntityId` /
`InvocableWithNodeVectors` concept hierarchy (see
`engine/system/include/irreden/system/component_system_event.hpp`).

For each parameter `P`:
- `std::is_lvalue_reference_v<P>` + `std::is_const_v<std::remove_reference_t<P>>`
  → `reads.insert(typeOf<std::remove_cvref_t<P>>())`.
- `std::is_lvalue_reference_v<P>` + non-const → `writes.insert(...)`.
- `std::is_same_v<std::remove_cvref_t<P>, IREntity::EntityId>` →
  `uses_entity_id = true`.
- Batch-form (parameter is `std::vector<T>&`) → record + set
  `is_batch_form = true`.

Tag types (`Spawns`, `Destroys`, `MainThread`, `AlsoReads<...>`,
`AlsoWrites<...>`, `ParallelSafe`) are detected by checking whether
the system's spec parameter pack contains them; they OR into the
access set.

## Acceptance criteria

- `engine/job/` builds clean on `linux-debug` and `macos-debug`.
- `IRJobs::parallelFor(0, 1024, 256, [](int, int){...})` smoke test:
  runs across all configured workers, returns when all complete.
- `IRJobs::isMainThread()` returns true on main, false inside
  `IRJobs::run` / `parallelFor` callbacks.
- `SystemAccess` unit tests cover all three tick signature forms +
  all tag compositions. Run via `IrredenEngineTest`.
- Apple Silicon startup logs the P-core cap decision.
- T-220's perf-gate matrix still passes (no perf regression from
  linking the lib).

## Gotchas

- Manager singletons (`g_entityManager`, etc.) are NOT accessed from
  workers in this PR. Don't pre-wrap them. T-225 owns the
  deferred-mutation surface.
- enkiTS uses pthreads on Unix; macOS/Linux symbols are fine. No
  Windows path to debug in this PR (the engine is OpenGL on Win and
  Metal on Mac, but threading lib is platform-neutral).
- The trait MUST be constexpr — runtime introspection of lambdas
  doesn't compose with the registration machinery. If the trait
  doesn't compile constexpr, the design has a hole; raise the issue
  before merging.
- Don't make `SystemAccess` reachable from creation-side code in this
  PR. Keep it internal to `engine/system/` until T-222 / T-224 give
  it consumers.

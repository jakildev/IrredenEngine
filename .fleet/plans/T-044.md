# Plan: T-044 — downstream auto-rebase when upstream changes

Part of the 5-task stacked-PR vision stack from issue #289.
See `.fleet/plans/T-041.md` for full plan and gotchas.

**Depends on:** T-043

**Files:**
- `scripts/fleet/fleet-claim` — add `molecule rebase-downstream` subcommand
- `.claude/commands/role-sonnet-author.md`
- `.claude/commands/role-opus-worker.md`

New subcommand reads molecule state, rebases each downstream branch onto the
new upstream tip in stack order, force-pushes with `--force-with-lease`.
Author role's "address feedback" sub-flow invokes this after pushing the
upstream fix and comments on each downstream PR.

Conflict handling: abort rebase, comment on affected PR, add `fleet:blocker`,
surface to human. Do NOT attempt heuristic resolution.

**Acceptance:** stack A→B→C; reviewer flags A with `fleet:needs-fix`; worker
fixes A, pushes, runs rebase subcommand; B and C branches now have new A tip
as parent. Conflicts surface as `fleet:blocker` + comment.
